ğŸ” Explanation: Foreign Key & Navigation Properties

ğŸ‘‰ Foreign Key (FK) = A field in one table that points to the Primary Key (PK) in another table.

In our case:

User.Id = Primary Key

Post.UserId = Foreign Key

ğŸ‘‰ Navigation Properties = The way EF Core lets you navigate between related entities.

User.Posts â†’ All posts written by that user (collection navigation).

Post.User â†’ The single user that owns this post (reference navigation).

ğŸ’¡ Without navigation properties, you would need to write manual SQL joins. With them, EF Core automatically loads relationships.

3. Create DbContext

Create file AppDbContext.cs:

using Microsoft.EntityFrameworkCore;
using EfCoreDemo.Entities;
namespace EfCoreDemo;
public class AppDbContext : DbContext
{
    public DbSet<User> Users { get; set; }
    public DbSet<Post> Posts { get; set; }

    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer(
            "Server=localhost;Database=EfDemo;Trusted_Connection=True;TrustServerCertificate=True"
        );
    }
    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<User>()
            .HasMany(u => u.Posts)
            .WithOne(p => p.User)
            .HasForeignKey(p => p.UserId);
    }
}
